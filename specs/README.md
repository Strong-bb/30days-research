# Specs

## 使用方法
1. 新任务 → 在 active/ 下创建 spec 文件
2. 用 AI 按 spec 实现
3. 完成 → 移到 archive/

## Spec 模板

复制以下模板到 active/ 下，填写后开始工作：

```markdown
# Spec: [功能名称]

## 目标
一句话描述要解决什么问题。

## 背景
- 相关的模块/文件（3-5 个）
- 依赖关系

## 实现方案
- 方案概述（不超过 5 步）
- 每步改什么文件、改什么

## 成功标准
- [ ] 验收条件 1
- [ ] 验收条件 2
- [ ] 相关测试通过

## 不做什么
- 明确排除的范围

## 风险
- 可能影响的其他模块
- 回滚策略
```

## 什么时候写 Spec
- 涉及 3 个以上文件的改动
- 新功能
- 重构
- 不确定怎么做时

## 什么时候不需要 Spec
- Bug 修复（1-2 个文件）
- 单文件小改动
- 文档更新
